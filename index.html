⚙️ Configuración Inicial — Supabase

Antes de usar el SGI por primera vez necesitás crear las tablas y el bucket de almacenamiento en tu proyecto Supabase (ahapckaovtgnxhwyuetw). Seguí estos 3 pasos una sola vez:

1

Crear las tablas en SQL Editor

Andá a Supabase → SQL Editor, creá una nueva query, pegá el siguiente SQL y ejecutalo:

-- SGI: Tablas del Sistema de Gestión Integrado -- Ejecutar en Supabase SQL Editor CREATE TABLE IF NOT EXISTS sgi_documentos ( id uuid PRIMARY KEY DEFAULT gen_random_uuid(), codigo text NOT NULL, nombre text NOT NULL, area text NOT NULL, tipo text DEFAULT 'Otro', estado text DEFAULT 'Vigente', fecha date, revision text, created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now() ); CREATE TABLE IF NOT EXISTS sgi_archivos ( id uuid PRIMARY KEY DEFAULT gen_random_uuid(), doc_id uuid REFERENCES sgi_documentos(id) ON DELETE CASCADE, filename text NOT NULL, storage_path text NOT NULL, size bigint, mime_type text, uploaded_at timestamptz DEFAULT now() ); -- Deshabilitar RLS para uso interno ALTER TABLE sgi_documentos DISABLE ROW LEVEL SECURITY; ALTER TABLE sgi_archivos DISABLE ROW LEVEL SECURITY;
2

Crear el bucket de almacenamiento

Andá a Supabase → Storage → New bucket y creá uno con estos datos exactos:

  • Name: sgi-docs
  • Public bucket: ✅ Activado (para poder ver y descargar archivos)
  • File size limit: 52428800 (50 MB)
3

Verificar la conexión

Hacé clic en "Probar conexión" para confirmar que todo está listo:

📥 Importar Documentos desde Excel
Instrucciones:
1. Usá la planilla template (columnas: codigo, nombre, area, tipo, estado, fecha, revision)
2. La primera fila con datos debe empezar en la fila 5 del template
3. Los campos obligatorios son: Código, Nombre y Área
